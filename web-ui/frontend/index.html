📖 Cursor Session Manager - 使用指南

🎯 功能说明

Cursor Session Manager 帮助您保存和恢复 Cursor Agent 的对话历史，让您可以在不同项目或工作状态之间自由切换。

💡 提示： 所有会话都是手动保存的，您需要在重要节点主动保存。未来版本将支持自动保存功能。

📝 如何保存会话？

1
在 Cursor 中完成工作
完成一个功能模块或到达重要里程碑后
2
点击"💾 保存当前会话"按钮
在本页面顶部找到保存按钮
3
输入会话名称和描述
例如："完成用户登录功能" / "修复UI颜色bug"
4
点击保存
会话将被保存，可以随时恢复

🔄 如何恢复会话？

1
找到要恢复的会话
在下方会话列表中找到您想恢复的会话卡片
2
点击"🔄 恢复"按钮
系统会自动备份当前会话，然后恢复选中的会话
3
完全关闭 Cursor
确保 Cursor 进程完全退出
4
重新打开 Cursor
打开 Cursor 并加载对应的项目
5
✅ 会话已恢复！
您将看到之前保存的完整对话历史
⚠️ 重要： 恢复会话后必须重启 Cursor才能生效！如果不重启，您将看不到恢复的对话历史。

🔍 如何快速找到会话？

当您有很多会话时，可以使用顶部的搜索框：

1
使用搜索框
在顶部搜索框输入关键词（会话名称或描述）
2
实时筛选
输入时会自动过滤显示匹配的会话
3
清空搜索
删除搜索内容即可显示所有会话

✏️ 如何管理会话？

每个会话卡片提供三个操作按钮：

  • 🔄 恢复： 将 Cursor 恢复到该会话的状态
  • ✏️ 编辑： 修改会话的名称和描述（不影响对话内容）
  • 🗑️ 删除： 永久删除该会话（无法恢复，请谨慎操作）

💡 最佳实践

1
定期保存重要节点
完成功能模块、解决重大问题后立即保存
2
使用描述性名称
清晰的名称和描述方便日后查找
3
项目切换前保存
切换项目前保存当前状态，方便回来继续
4
定期清理旧会话
删除不再需要的旧会话，保持列表整洁

🎯 典型使用场景

场景1：多项目工作

项目A工作 → 保存会话"项目A功能开发" → 切换到项目B → 需要回项目A → 搜索并恢复"项目A功能开发" → 重启Cursor → 继续之前的工作

场景2：实验性开发

保存稳定版本 → 尝试新方案 → 如果不满意 → 恢复到稳定版本 → 重新开始

场景3：重要里程碑

完成核心功能 → 保存"核心功能完成" → 继续其他开发 → 以后需要回顾 → 恢复该会话查看当时的讨论

❓ 常见问题

Q: 恢复会话后看不到对话历史？

A: 确保完全关闭 Cursor（退出进程），然后重新打开。

Q: 会话会自动保存吗？

A: 目前是手动保存，需要您主动点击保存按钮。未来版本将支持自动保存。

Q: 删除的会话能恢复吗？

A: 无法恢复，删除是永久性的，请谨慎操作。

Q: 我有很多会话，如何管理？

A: 使用搜索功能快速查找，定期清理不需要的旧会话。
